# include_end_by
:::warning 
Свойство работает только в устаревшем расширении для повторяющихся событий.
:::
### Description

@short: Определяет, должна ли дата, указанная в поле 'End by', быть исключительной или включительной

@signature: include_end_by: boolean

### Example

~~~jsx
scheduler.config.include_end_by = true;
...
scheduler.init('scheduler_here', new Date(2027, 7, 5), "week");
~~~

**Значение по умолчанию:** false

### Details

:::note
Свойство требует включённого расширения [recurring](guides/extensions-list.md#recurring).
:::

По умолчанию дата, указанная в поле 'End by', исключается.

Например, если пользователь укажет значение '01.15.2027' в поле 'End by', то:

- если <code>include_end_by = false</code> (значение по умолчанию) — серия повторяющихся событий завершится на 01.14.2027.
- Если <code>include_end_by = true</code> — серия повторяющихся событий завершится на 01.15.2027.

## How does the database save the dates?

Все даты, выбираемые в scheduler, имеют часовую и минутную часть, то есть дата *15.11.2027*, выбранная в любом date picker, будет интерпретирована как *15.11.2027 00:00*.

Это повлияет на продолжительность серии, когда вы выбираете поле 'End by' в форме повторения.

Например, если пользователь указывает значение *15.11.2027* в поле 'End by', тогда:

- если <code>include_end_by = false</code> (значение по умолчанию) — конечная дата повторяющейся серии будет сохранена как *15.11.2027 00:00*, и последний возможный экземпляр серии может произойти до *14.11.2027 23:59*, поэтому в выбранный день в серия не будет иметь событий;
- если <code>include_end_by = true</code> — конечная дата повторяющейся серии будет сохранена как *16.11.2027 00:00* (полночь после выбранной даты), поэтому выбранный день будет включён в продолжительность серии и последний экземпляр серии может произойти до *15.11.2027 23:59*.
