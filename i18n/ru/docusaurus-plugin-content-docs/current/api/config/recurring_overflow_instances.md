# recurring_overflow_instances
:::warning 
Свойство работает только в устаревшем extension для повторяющихся событий
:::
### Description

@short: Определяет поведение повторов, которые переходят к следующему месяцу

@signature: recurring_overflow_instances: string

### Example

~~~jsx
scheduler.config.recurring_overflow_instances = "lastDay";
~~~

**Значение по умолчанию:** undefined

### Details

:::note
 Свойство требует включенного extension [recurring](guides/extensions-list.md#recurring). 
:::

Возьмем событие, которое происходит каждый месяц 30-го числа, и рассмотрим его поведение в феврале для каждого варианта:

- **"skip"** - пропускает событие, которое приходится на дату, которая не существует. *Событие будет пропущено в феврале.* 
- **"lastDay"** - перемещает событие, происходящее в несуществующую дату, на последний день месяца. *Событие произойдет 28 февраля.* 
- **"none"** - перемещает событие, происходящее в несуществующую дату, на первый день следующего месяца. *Событие произойдет 1 марта.*

Если опция не определена, будет применено поведение "skip".
