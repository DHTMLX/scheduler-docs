# DHTMLX MCP Server

ИИ-ассистенты по кодированию, такие как Claude, Cursor или ChatGPT, могут создавать устаревший или неточный код при работе с API, специфичными для библиотек. Сервер DHTMLX MCP устраняет это, предоставляя прямой доступ к текущей документации и справке по API.

## Что такое MCP

Model Context Protocol (MCP) — это стандарт, который предоставляет AI-помощникам внешний контекст по конкретным инструментам и библиотекам.

Большие языковые модели обучаются на данных до определённой даты и не отражают автоматически недавние изменения API или появление новых функций. Сервер DHTMLX MCP заполняет этот пробел, предоставляя полную и актуальную документацию через систему RAG (Retrieval-Augmented Generation).

**URL сервера:** `https://docs.dhtmlx.com/mcp`

:::note
Сервер DHTMLX MCP — это общая служба, охватывающая все основные продукты DHTMLX, а не только Scheduler. Инструкции по настройке в этом разделе применимы независимо от того, с каким компонентом DHTMLX вы работаете.
:::

## Поддерживаемые продукты

При подключении инструмент ИИ может получать документацию, генерировать фрагменты кода на основе текущих API и отвечать на вопросы по настройке для следующих продуктов:

- Gantt
- Scheduler
- Suite (грид, Form, TreeGrid, и прочие)
- Kanban
- Pivot
- Spreadsheet
- Diagram
- RichText

## Конфиденциальность

Сервер MCP — размещённая услуга. Он не работает локально и не получает доступ к вашим файлам. Личная информация о пользователях не хранится. Запросы могут регистрироваться для отладки и улучшения сервиса. Для коммерческих вариантов с строгой политикой отсутствия логирования свяжитесь с `info@dhtmlx.com`.

## Настройка

Выберите ваш инструмент ИИ ниже и следуйте соответствующим инструкциям.

### Claude Code

Рекомендуемый способ — через CLI:

```bash
claude mcp add --transport http dhtmlx-mcp https://docs.dhtmlx.com/mcp
```

Либо добавьте следующее в файл `mcp.json` вручную:

```json
{
  "mcpServers": {
    "dhtmlx-mcp": {
      "type": "http",
      "url": "https://docs.dhtmlx.com/mcp"
    }
  }
}
```

### Cursor

1. Откройте настройки (Cmd+Shift+J на Mac, Ctrl+Shift+J на Windows/Linux)
2. Перейдите в **Tools & MCP**
3. Нажмите **Add Custom MCP**
4. Вставьте следующую конфигурацию:

```json
{
  "mcpServers": {
    "dhtmlx-mcp": {
      "url": "https://docs.dhtmlx.com/mcp"
    }
  }
}
```

После подключения можно напрямую в чате использовать подсказки вроде "Проверьте документацию DHTMLX для настройки повторяющихся событий в Scheduler".

### Gemini CLI

Откройте файл конфигурации по адресу `~/.gemini/settings.json` и добавьте:

```json
{
  "mcpServers": {
    "dhtmlx-mcp": {
      "url": "https://docs.dhtmlx.com/mcp"
    }
  }
}
```

Перезапустите Gemini CLI, чтобы применить изменения.

### Antigravity (Google)

1. Откройте палитру команд
2. Введите "mcp add"
3. Выберите "HTTP"
4. Введите URL: `https://docs.dhtmlx.com/mcp`
5. Введите имя: `dhtmlx-mcp`

### Другие инструменты

Большинство современных инструментов ИИ для кодирования поддерживают MCP через настройки. Ищите "Model Context Protocol", "Context Sources" или аналогичный параметр и добавьте `https://docs.dhtmlx.com/mcp` в качестве пользовательского источника.

### ChatGPT

Обратите внимание, что интеграция MCP с ChatGPT может привести к более медленному времени отклика (примерно 20 секунд на запрос). Для более быстрого опыта рассмотрите возможность использования одного из вышеупомянутых инструментов.

Чтобы настроить ChatGPT:

1. Перейдите в **Settings** → **Apps & Connectors**
2. Нажмите **Advanced settings**
3. Включите **Developer mode**
4. Вернитесь на экран Connectors и нажмите кнопку **Create**
5. Заполните следующее:
   - Name: `dhtmlx-mcp`
   - URL: `https://docs.dhtmlx.com/mcp`
   - Authentication: `No authentication`
6. Нажмите **Create**

После настройки вы можете попросить ChatGPT обращаться к серверу DHTMLX MCP при работе с любым компонентом DHTMLX.

## Советы для достижения наилучших результатов

При формулировании запроса явно используйте документацию DHTMLX для более точных результатов. Например:

- "Используя документацию DHTMLX, как изменить шкалу времени в Scheduler?"
- "Проверьте DHTMLX MCP для конфигурации повторяющихся событий в Scheduler"

Чем точнее запрос, тем точнее будет результат.
