# Соответствие Content Security Policy

:::warning
 Данный функционал устарел и больше не поддерживается.
:::
Content Security Policy (CSP) - это веб-стандарт, предназначенный для предотвращения запуска неавторизованного JavaScript-кода.

Библиотека dhtmlxScheduler включает расширение **ext/dhtmlxscheduler_csp.js**, которое помогает dhtmlxScheduler корректно работать при включённой CSP в вашем приложении. Это расширение повышает безопасность ваших приложений.

Чтобы включить поддержку CSP в приложении, созданном с помощью Scheduler, просто добавьте *dhtmlxscheduler_csp.js* сразу после *dhtmlxscheduler.js*:

~~~html
<script src="../codebase/ext/dhtmlxscheduler_csp.js"></script>
~~~

Расширение *dhtmlxscheduler_csp.js* работает поверх базового Scheduler и заменяет небезопасный код (в основном, форматирование и парсинг дат). Обратите внимание, что необходимо разрешить использование inline-стилей, так как они применяются во многих частях компонента.

В некоторых случаях CSP-дружелюбные методы могут немного снизить производительность, поэтому они не включены по умолчанию и должны быть активированы вручную.

:::warning
 Эта возможность устарела начиная с версии 6.0. Вместо этого настройте режим для внутренней реализации методов форматирования дат. [Подробнее](api/config/csp.md)
:::
